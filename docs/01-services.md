# Этап 1: Описание сервисов

## Обзор системы

Dating Bot — телеграм-бот для знакомств. Пользователи заполняют анкету, просматривают других пользователей (лайк/пас), при взаимном лайке получают мэтч и возможность общения.

---

## 1. Сервис бота (Bot Service)

**Назначение:** Единственная точка входа пользователя в систему.

**Функции:**
- Интеграция с Telegram Bot API (получение обновлений, отправка сообщений)
- Обработка команд: `/start`, `/help`, навигация по меню
- Оркестрация сценариев: регистрация, заполнение/редактирование анкеты, просмотр анкет (свайпы), чат после мэтча
- Отправка уведомлений (новый мэтч, новое сообщение)
- Валидация ввода и перевод пользователя между состояниями (FSM)

**Входы:** Webhook или long polling от Telegram.  
**Выходы:** Запросы к сервису пользователей, сервису анкет, сервису ранжирования, сервису мэтчей/чатов.

**Технологии:** Python (aiogram / python-telegram-bot), асинхронная обработка.

---

## 2. Сервис пользователей (User Service)

**Назначение:** Учёт пользователей и аутентификация.

**Функции:**
- Регистрация по Telegram ID (при первом `/start`)
- Хранение привязки Telegram ID ↔ внутренний user_id
- Базовые данные: дата регистрации, последняя активность, статус (активен/заблокирован)
- Предоставление данных пользователя другим сервисам по user_id

**Входы:** Запросы от Bot Service (создать пользователя, получить по telegram_id).  
**Выходы:** Ответы с user_id и флагами (новый/существующий пользователь).

**Хранение:** Таблицы `users` в БД.

---

## 3. Сервис анкет (Profile Service)

**Назначение:** CRUD анкет и медиа.

**Функции:**
- Создание, чтение, обновление анкеты пользователя (имя, возраст, пол, город, описание, интересы)
- Загрузка и привязка фотографий к анкете (хранение метаданных; файлы — в S3/Minio)
- Проверка полноты анкеты (для первичного рейтинга)
- Выдача анкеты по ID для отображения в боте

**Входы:** Запросы от Bot Service, от сервиса ранжирования (получить анкету по ID).  
**Выходы:** Объект анкеты с полями и списком фото.

**Хранение:** Таблицы `profiles`, `profile_photos`; файлы — объектное хранилище (S3/Minio).

---

## 4. Сервис ранжирования (Ranking Service)

**Назначение:** Выдача пользователю подходящих анкет в нужном порядке.

**Функции:**
- **Первичный рейтинг:** учёт данных анкеты (возраст, пол, интересы, город), полноты анкеты, количества фото, предпочтений по возрасту/полу/городу
- **Поведенческий рейтинг:** учёт лайков/пасов, мэтчей, инициаций диалогов, активности по времени
- **Комбинированный рейтинг:** весовая модель первичный + поведенческий (опционально рефералы)
- Формирование и обновление очереди анкет для пользователя
- Интеграция с кэшем (Redis): предзагрузка N анкет в кэш при старте сессии, пополнение по мере просмотра

**Входы:** user_id, предпочтения, события (лайк/пас/мэтч) от Bot Service или через MQ.  
**Выходы:** Список ID анкет (или готовых объектов) для показа в боте.

**Хранение:** Таблицы рейтингов, кэш в Redis.

---

## 5. Сервис взаимодействий (Interaction Service)

**Назначение:** Лайки, пасы, мэтчи.

**Функции:**
- Фиксация лайка/паса (кто → кого)
- Определение мэтча (взаимный лайк)
- Создание записи мэтча и открытие чата
- Предоставление истории действий для сервиса ранжирования и для отображения в боте

**Входы:** Запросы от Bot Service (лайк/пас), события от других сервисов.  
**Выходы:** Факт мэтча, список мэтчей пользователя, список лайков/пасов.

**Хранение:** Таблицы `likes`, `passes`, `matches`.

---

## 6. Сервис чатов (Chat Service)

**Назначение:** Сообщения между пользователями после мэтча.

**Функции:**
- Отправка сообщения в рамках мэтча
- История переписки по паре пользователей
- Уведомление о новом сообщении (для Bot Service)

**Входы:** Запросы от Bot Service (отправить сообщение, получить историю).  
**Выходы:** Сохранённые сообщения, список диалогов с последним сообщением.

**Хранение:** Таблицы `match_chats`, `messages`.

---

## Вспомогательные компоненты (на последующих этапах)

| Компонент   | Назначение                                      |
|------------|--------------------------------------------------|
| **Redis**  | Кэш предранжированных анкет, сессии, очереди     |
| **Celery** | Отложенные задачи: пересчёт рейтингов по расписанию |
| **MQ** (Kafka/RabbitMQ) | Поток событий: лайки, пасы, мэтчи — для ранжирования и аналитики |
| **S3/Minio** | Хранение фотографий анкет                      |

---

## Взаимодействие на первом этапе

```
[Telegram] ←→ [Bot Service] ←→ [User Service]
                    ↓
              [Profile Service]
                    ↓
              [Ranking Service]  (позже + Redis)
                    ↓
              [Interaction Service] → [Chat Service]
```

На этапе 1 достаточно спроектировать эти сервисы, БД и общую архитектуру; реализация Bot + User + минимальный Profile — на этапе 2.
